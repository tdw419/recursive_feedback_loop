## Review Criteria
For each issue found:
1. Specific section or paragraph
2. What's wrong (ambiguous, inconsistent, missing, or wrong)
3. Concrete suggestion for fixing it

Focus areas:
- Ambiguity: Could two engineers interpret this differently?
- Completeness: Are error cases covered? Edge cases? Backward compatibility?
- Consistency: Do different sections contradict each other?
- Feasibility: Can this actually be built as described?
- Missing pieces: What's not mentioned that would be needed?

Iteration approach:
- Iteration 1: Surface-level review, flag obvious issues
- Iteration 2+: Go deeper — find issues that only appear when you think about implementation details
- Each iteration should find NEW issues, not restate old ones
